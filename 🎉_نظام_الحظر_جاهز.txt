الخطوة 1️⃣: احصل على GitHub Token
────────────────────────────────

1. روح لـ: https://github.com/settings/tokens
2. اضغط "Generate new token (classic)"
3. اسم: License Server Kill Switch
4. اختر: repo (✅ كل شي)
5. Generate token
6. انسخ Token (يظهر مرة واحدة فقط!)


الخطوة 2️⃣: حفظ Token
───────────────────

روح لمجلد: server/

اصنع ملف جديد: github_token.txt

افتحه والصق Token فيه:
ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

احفظ وأغلق


الخطوة 3️⃣: شغّل السيرفر
────────────────────────

افتح CMD في مجلد server:

cd "j:\New folder (5)\smart_x_bot\server"
python app.py

✅ السيرفر شغال!

💡 كيف يشتغل:
═══════════════════════════════════════════════════════════════════

عند الحظر:
──────────
1. Admin يضغط "Block"
2. السيرفر يجيب Hardware ID من قاعدة البيانات
3. يجلب blacklist.json من GitHub
4. يضيف Hardware ID للقائمة
5. يرفع التحديث لـ GitHub
6. GitHub يحفظ التغييرات
7. خلال دقيقتين: المستخدم البرنامج يسكر عنده!


عند المستخدم:
─────────────
1. يفتح البرنامج
2. البرنامج يفحص blacklist.json من GitHub
3. يلاقي Hardware ID تبعه في القائمة
4. يطلعله رسالة: "خطأ في الاتصال بالخادم"
5. البرنامج يسكر تلقائياً
6. المستخدم ما يعرف إنه محظور! 😎

───────────────────────────────────

───────────────────────────────────────────
الحل:
• تأكد Token صحيح
• تأكد Token عنده صلاحية repo
• تأكد المستودع موجود وعام (Public)


مشكلة: المستخدم لسه يستخدم البرنامج
──────────────────────────────────────
الحل:
• انتظر 2 دقيقة (GitHub يحتاج وقت)
• المستخدم لازم يعيد تشغيل البرنامج
• أو ينتظر الفحص التلقائي (كل ساعة)
